# 公众号文章风格分析与模仿系统（v3.0）

## 适用场景（何时调用）

- 用户希望模仿某位公众号作者写作风格，并已准备 3-5 篇同一作者的文章样本
- 用户希望得到“可复用的风格模型 + 模仿策略 + 结构化输出 JSON”

## 输入约定

- 样本目录：项目根目录下 `imitation_article/`
  - 支持：`.md`、`.txt`
  - 建议：3-5 篇，同一作者
- 输出目录：项目根目录下 `style/`
- 输出文件名：`style/<作者名>文章风格分析与模仿系统.md`

## 执行流程（必须按顺序）

1. **发现样本**
   - 若 `imitation_article/` 不存在则创建并提示用户放入文章后再执行
   - 读取目录下全部 `.md`/`.txt`，按文件名排序
   - 若少于 2 篇，仍可分析但需在结果中标注“稳定性评估受限”

2. **确定作者名**
   - 优先从样本文本中提取可能的作者署名（例如末尾“作者：XXX”“— XXX”“By XXX”）
   - 若无法可靠提取：使用 `未知作者`
   - 输出文件名必须使用最终确定的作者名

3. **逐篇分析（单篇分析）**
   - 对每篇文章做“宏观风格 + 微观风格”两层分析
   - 记录可复现的证据点：高频句式、标志性表达、结构套路、过渡方式、情绪基调、常用意象
   - 避免编造：对没有在样本中出现的风格特征不要输出为确定结论

4. **交叉对比（跨篇对比）**
   - 找到多篇稳定重复的特征，并明确“恒定特征 vs 主题特定变化”
   - 识别不同主题/体裁下作者的“适应性变化方式”，但必须回溯到样本证据

5. **构建风格模型并输出模仿策略**
   - 产出“可执行的模仿指南”，包括模仿优先级、步骤、主题迁移策略、常见误区
   - 明确“如何写”而非仅“是什么”

6. **落盘输出**
   - 确保 `style/` 存在，不存在则创建
   - 生成 `style/<作者名>文章风格分析与模仿系统.md`
   - 文档正文以一个 JSON 代码块为主（必须与下方结构一致），必要时可在 JSON 前加一段简短说明（不超过 120 字）

## 生成提示词（严格照此执行）

你将对 `imitation_article/` 中同一作者的多篇文本样本进行综合分析，提取作者稳定的风格特征并提供从宏观到微观的全面模仿指南。

### 分析原则

1. **多篇交叉验证**：通过分析多篇文章识别稳定出现的风格特征
2. **共性与特性区分**：区分作者的"恒定风格特征"与"特定主题下的变化特征"
3. **宏微观结合**：既分析思维模式与表达意图，又分析具体语言与技巧特征
4. **风格适应性**：分析作者如何在不同主题中调整风格同时保持个人特色

### 分析流程

1. **单篇分析**：对每篇文章进行宏观和微观风格分析
2. **交叉对比**：对比多篇文章找出共性和差异
3. **提取恒定特征**：确定在不同文章中保持一致的风格元素
4. **识别适应性特征**：分析作者如何根据不同主题调整风格
5. **风格模型构建**：建立该作者的风格模型和模仿策略

### 分析维度

#### 宏观风格维度
1. 思维模式（逻辑结构、思考方式、认知框架）
2. 价值取向（作者立场、评价标准、关注重点）
3. 表达意图（写作目的、预期效果、读者关系）
4. 风格定位（文体归属、风格流派、典范参照）
5. **跨文本一致性**（不同主题下保持的核心风格元素）

#### 微观风格维度
1. 语言特征（句式、用词、修辞）
2. 结构特征（段落、过渡、层次）
3. 叙事特征（视角、距离、时序）
4. 情感特征（浓淡、方式、基调）
5. 思维特征（逻辑、深度、节奏）
6. 个性标记（独特表达、意象系统）
7. 文化底蕴（典故、知识领域）
8. 韵律节奏（音节、停顿、节奏）
9. **跨文本变化**（不同主题下的风格调整方式）
